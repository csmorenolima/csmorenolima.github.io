Apple Shortcuts automation

Daily prayer delivery, fully automated.

Morning Prayer Bot is a lightweight iPadOS workflow that monitors a YouTube prayer feed, sends a motivational message, and shares the latest daily video to a WhatsApp group.

View workflow See architecture

Project snapshot

A focused automation for one recurring daily task.

The system replaces a repetitive morning routine with a small, understandable workflow that can run without a custom server or complex infrastructure.

Platform
iPadOS
Frequency
Daily
Input
YouTube RSS
Processing
RSS2JSON + Shortcuts
Output

The problem

A simple task that still had to be remembered every morning.

Sharing the latest prayer video involved checking the channel, finding the newest upload and posting it manually at a consistent time. The process was repetitive and could easily be delayed or forgotten.

The response

Automate retrieval, validation and delivery.

A scheduled Apple Shortcut retrieves the newest feed item, extracts the required metadata, checks whether the video has already been sent and posts only new content to WhatsApp.

System workflow

From YouTube upload to WhatsApp delivery.

The sequence uses structured feed data and a small amount of stored state to ensure that only the newest unsent video reaches the group.

Core capabilities

Four functions keep the workflow useful and dependable.

The project deliberately limits its scope. Each function directly supports reliable daily delivery rather than adding unnecessary infrastructure.

Scheduled retrieval

The Shortcut runs at a consistent time without requiring a manual check.

Feed parsing

RSS2JSON creates a predictable structure that Shortcuts can process reliably.

Duplicate prevention

A stored video reference stops repeated delivery across scheduled runs.

Direct delivery

The latest valid link is shared through the familiar WhatsApp channel.

Architecture

A mobile-first pipeline with minimal moving parts.

The implementation keeps orchestration, validation and delivery on the iPad while relying on established services for the feed and messaging layers.

01

Structured input

YouTube RSS is converted before it reaches Shortcuts, reducing parsing complexity inside the automation.

02

Local state

The last delivered video is stored locally, giving the Shortcut a simple and persistent duplicate check.

03

No custom backend

The workflow remains understandable and maintainable without hosting, databases or server-side code.

Engineering the workflow

Technical decisions shaped by reliability and simplicity.

Data handling

Convert the feed before parsing it.

RSS2JSON reduces the amount of XML handling required inside Apple Shortcuts and exposes only the structured values needed by the workflow.

State management

Store the last delivered item locally.

Comparing the latest video with a saved reference creates duplicate protection without introducing a database or account system.

System scope

Keep orchestration on the device.

A mobile-first build makes the system easier to inspect, adapt and maintain from the iPad that already runs the daily automation.

Challenges and fixes

Making a small workflow dependable.

Reliability depended less on adding tools and more on controlling input, state and failure points.

Challenge

Feed data included more information than the workflow required.

Response

Extract only the title, link, publication date and thumbnail before continuing.

Challenge

Repeated scheduled runs could post the same video more than once.

Response

Compare each feed item against a locally stored reference to the previously delivered video.

Challenge

Apple Shortcuts required predictable data structures to avoid broken runs.

Response

Convert the RSS feed into JSON before parsing and accessing individual values.

Challenge

The project needed to stay lightweight and manageable without a server.

Response

Use existing services and local device automation rather than building custom infrastructure.

Future development

Focused improvements for the next version.

Further development would concentrate on observability and configuration rather than making the core workflow more complex.

Project outcome

A reliable daily sharing system with minimal overhead.

Morning Prayer Bot demonstrates how a small, carefully scoped automation can remove a repetitive manual task while remaining understandable and easy to maintain.
